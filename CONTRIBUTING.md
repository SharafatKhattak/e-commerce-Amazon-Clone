# Contributing Guide

Thank you for your interest in contributing to the Amazon Clone project!

## 🤝 How to Contribute

### 1. Fork the Repository
```bash
git clone https://github.com/your-username/ecommerce-amazon-clone.git
cd ecommerce-amazon-clone
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow the existing code style
- Write clean, readable code
- Add comments for complex logic
- Test your changes

### 4. Commit Your Changes
```bash
git commit -m "feat: add your feature description"
```

Use these prefixes:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation update
- `style:` - Code style changes
- `refactor:` - Code refactoring
- `test:` - Testing updates

### 5. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 6. Create a Pull Request
- Describe your changes clearly
- Reference related issues
- Include screenshots for UI changes

## 📋 Code Standards

### JavaScript/TypeScript
- Use TypeScript for type safety
- Follow ESLint rules
- Use functional components in React
- Implement proper error handling

### Git Commits
- Keep commits focused and atomic
- Write clear commit messages
- Reference issues when applicable

### Testing
- Test new features locally
- Ensure no breaking changes
- Test responsive design

## 🐛 Bug Reports

Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment details

## 💡 Feature Requests

Describe:
- The feature you'd like to see
- Why it's useful
- Suggested implementation (optional)

## 📖 Documentation

Help us improve documentation by:
- Fixing typos and errors
- Clarifying confusing sections
- Adding examples
- Improving API documentation

## 🎨 Design Contributions

- Maintain consistency with Amazon's design
- Ensure accessibility (WCAG 2.1 AA)
- Test on multiple devices
- Optimize images

## 🧪 Testing Contributions

- Write unit tests for components
- Write integration tests for APIs
- Test edge cases
- Test error scenarios

## 📦 Adding Dependencies

- Discuss major dependencies in issues first
- Choose packages with good maintenance
- Prefer npm over unpkg imports
- Keep bundle size in mind

## 🚀 Performance

- Optimize images
- Minimize bundle size
- Use code splitting
- Implement lazy loading

## ♿ Accessibility

- Test with screen readers
- Ensure keyboard navigation
- Provide alt text for images
- Use semantic HTML

## 📝 Code Comments

Bad:
```javascript
// increment i
i++;
```

Good:
```javascript
// Move to next product in pagination
currentPage++;
```

## ✅ Before Submitting PR

- [ ] Code follows style guidelines
- [ ] Self-review of changes
- [ ] Comments added for complex logic
- [ ] No new warnings generated
- [ ] Tests added/updated
- [ ] Documentation updated
- [ ] No breaking changes

## 🏆 Recognition

Contributors will be recognized in:
- README.md
- GitHub Contributors page
- Release notes

---

Thank you for contributing! 🎉
